## ✅ Penjelasan Alur Algoritma — Latihan 1 (latihan1.py)

Program menampilkan n bilangan acak yang nilainya kurang dari 0.5

## Alur Algoritma

1. Program meminta input nilai n dari pengguna (jumlah data yg ingin ditampilkan).

2. Mengimport fungsi random dari modul random.

3. Membuat perulangan sebanyak n kali.

4. Di setiap perulangan:

    *Menghasilkan angka acak antara 0 dan 1.

    *Mengecek apakah angka tersebut kurang dari 0.5.

    *Jika kurang dari 0.5 → tampilkan angka tersebut.

5. Setelah selesai, tampilkan pesan bahwa proses telah selesai.

## Tujuan

Mengetahui cara menggunakan fungsi random() dan memfilter nilai tertentu menggunakan perulangan.

## ✅ Penjelasan Alur Algoritma — Latihan 2 (latihan2.py)

Program menghitung total laba usaha selama 8 bulan

## Alur Algoritma

1. Menetapkan modal awal 100 juta rupiah.

2. Membuat variabel untuk menampung total laba (di awal bernilai 0).

3. Melakukan perulangan dari bulan 1 sampai bulan 8.

4. Setiap bulan, tentukan persentase laba:
   

    *Bulan 1–2 → belum ada laba (0%)

    *Bulan 3–4 → laba 1% dari modal

    *Bulan 5–7 → laba 5% dari modal

    *Bulan 8 → laba turun menjadi 3%

5. Hitung laba bulan tersebut.

6. Tambahkan laba bulan itu ke total laba.

7. Tampilkan laba tiap bulan.

8. Setelah perulangan selesai, tampilkan total laba selama 8 bulan.

## Tujuan

Melatih logika percabangan dan perulangan dalam menghitung besaran laba tiap periode.

## ✅ Penjelasan Alur Algoritma — Latihan 3 (latihan3.py)

Program simulasi ATM sederhana (tarik uang)

## Alur Algoritma

1. Menetapkan saldo awal = Rp 1.000.000.

2. Menampilkan menu ke pengguna:
   

    1 = Tarik uang

    2 = Keluar

4. Meminta pilihan pengguna.

5. Jika pengguna memilih 1 (Tarik uang):
   

    *Meminta jumlah uang yang ingin ditarik.

    *Mengecek apakah saldo cukup.

    *Jika cukup → kurangi saldo dan tampilkan saldo terbaru.

    *Jika tidak cukup → tampilkan pesan saldo tidak cukup.

    *Kembali ke menu selama saldo masih ada.

5. Jika pengguna memilih 2 (Keluar):

    *Tampilkan pesan terima kasih dan program berhenti.

6. Program berhenti jika saldo habis atau user memilih keluar.

## Tujuan

Memahami penggunaan:

  Perulangan

  Percabangan

  Operasi aritmatika pada saldo rekening
